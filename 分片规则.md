# 音符分片规则

## 1. 基本对象

### 1.1 音符

一个音符 (n) 具有以下属性：

* (n.position\in[0,+\infty))；
* (n.length\in(0,+\infty))；
* (n.lyric) 为字符串；
* (n) 依附有有限个音素。

定义：

[
s(n)=n.position
]

[
e(n)=n.position+n.length
]

音符 (n) 所表示的时间区间为左闭右开区间：

[
I(n)=[s(n),e(n))
]

### 1.2 音素

依附于音符 (n) 的每个音素 (q) 具有：

* (q.start\in\mathbb{R})；
* (q.onset\in{\mathrm{true},\mathrm{false}})。

音素的绝对位置为：

[
s(n)+q.start
]

音素的绝对位置不直接参与分片边界计算；属性 `start` 仅用于确定音素的排列顺序。

### 1.3 分片

一个分片 (p) 具有：

* (p.position\in\mathbb{R})；
* (p.length\in(0,+\infty))。

定义：

[
s(p)=p.position
]

[
e(p)=p.position+p.length
]

分片 (p) 所表示的时间区间为：

[
I(p)=[s(p),e(p))
]

### 1.4 预设参数

给定：

* 休止符谓词 (R(x))；
* 填充基础量 (B>=0)；
* 填充附加量 (A>=0)；
* 分片间距 (D>=0)。

其中，(R(n.lyric)=\mathrm{true}) 表示音符 (n) 为休止符。

---

## 2. 音符的派生填充量

### 2.1 前导非 onset 音素数量

对于音符 (n)，将其全部音素按照 `start` 从小到大排列，得到序列：

[
q_1,q_2,\ldots,q_m
]

若存在相同的 `start`，则这些音素之间的顺序可以任意确定。

定义 (c(n)) 为该序列中满足 `onset = false` 的最长连续前缀的音素数量，即：

[
c(n)=\max\left{
k\in{0,\ldots,m}
\mid
\forall i\in{1,\ldots,k},\ q_i.onset=\mathrm{false}
\right}
]

特别地：

* 若音符不包含音素，则 (c(n)=0)；
* 若第一个音素满足 `onset = true`，则 (c(n)=0)。

### 2.2 左填充量

定义音符 (n) 的左填充量为：

[
L(n)=
\begin{cases}
0, & R(n.lyric)=\mathrm{true}[4pt]
\min(B+c(n)A,\ D), & R(n.lyric)=\mathrm{false}
\end{cases}
]

### 2.3 右填充量

定义音符 (n) 的右填充量为：

[
U(n)=
\begin{cases}
0, & R(n.lyric)=\mathrm{true}[4pt]
B, & R(n.lyric)=\mathrm{false}
\end{cases}
]

---

## 3. 音符集合所诱导的分片

设 (S) 是一个非空音符集合。

从 (S) 中选取一个起始位置最小的音符：

[
n_L(S)\in\arg\min_{n\in S}s(n)
]

从 (S) 中选取一个结束位置最大的音符：

[
n_R(S)\in\arg\max_{n\in S}e(n)
]

若存在多个并列音符，则可以任取一个。

定义：

[
S.note_start=s(n_L(S))
]

[
S.note_end=e(n_R(S))
]

音符集合 (S) 所诱导的唯一候选分片记为 (P(S))，其边界为：

[
s(P(S))=S.note_start-L(n_L(S))
]

[
e(P(S))=S.note_end+U(n_R(S))
]

因此：

[
P(S).position=S.note_start-L(n_L(S))
]

[
P(S).length=
S.note_end+U(n_R(S))
-\bigl(S.note_start-L(n_L(S))\bigr)
]

由于 (S) 非空且每个音符长度为正，因此：

[
P(S).length>0
]

并且对于任意 (n\in S)，均有：

[
I(n)\subseteq I(P(S))
]

一个分片的边界只能按照上述方式由其包含的音符集合确定，不得额外扩大或缩小。

---

## 4. 合法分片结果

设输入音符集合为有限集合 (N)。

一个分片结果由以下两部分组成：

1. 对音符集合 (N) 的一个划分

[
\mathcal S={S_1,S_2,\ldots,S_k}
]

2. 每个子集 (S_i) 所诱导的分片

[
p_i=P(S_i)
]

其中，划分必须满足：

[
S_i\ne\varnothing
]

[
S_i\cap S_j=\varnothing
\qquad(i\ne j)
]

[
\bigcup_{i=1}^{k}S_i=N
]

也就是说，每个分片必须包含至少一个音符，并且每个输入音符必须且只能属于一个分片。

### 4.1 分片互不重叠

对于任意两个不同分片 (p_i,p_j)，必须满足：

[
I(p_i)\cap I(p_j)=\varnothing
]

由于分片采用左闭右开区间，允许一个分片的右边界等于另一个分片的左边界。

### 4.2 分片的时间顺序

将全部分片按照 `position` 从小到大排列：

[
s(p_1)<s(p_2)<\cdots<s(p_k)
]

则对于任意相邻分片 (p_i,p_{i+1})，必须满足：

[
e(p_i)\le s(p_{i+1})
]

### 4.3 音符主体间距

对于任意相邻分片 (p_i,p_{i+1})，必须满足：

[
p_{i+1}.note_start-p_i.note_end\ge D
]

该条件等价于原规则中关于前驱分片的要求：对于分片 (p_{i+1})，(p_i) 是所有满足

[
e(p)\le s(p_{i+1})
]

的分片中右边界最大的分片。

第一个分片不存在前驱，因此不受此条件约束。

---

## 5. 最优性

设所有满足第 4 节要求的合法分片结果组成集合 (\mathcal F)。

对于合法结果 (F\in\mathcal F)，记其中的分片数量为：

[
|F|
]

最终分片结果 (F^\ast) 必须满足：

[
|F^\ast|
========

\max_{F\in\mathcal F}|F|
]

即最终结果必须在全部合法分片结果中具有最多的分片数量。

本规则中的“分片长度尽可能小”严格解释为“分片数量最多”，不表示：

* 分片总长度最小；
* 单个分片长度分别最小；
* 填充长度之和最小。

除最大化分片数量外，不设置其他次级优化目标。

若输入音符集合为空，则输出为空分片集合。

---

## 6. 等价的顺序扫描算法

### 6.1 音符排序

将输入音符按照起始位置从小到大排列：

[
s(n_1)\le s(n_2)\le\cdots\le s(n_m)
]

若起始位置相同，则顺序可以任意确定。

算法从左到右扫描该序列，并维护当前尚未输出的音符组 (S)。

### 6.2 候选切分条件

假设当前音符组为：

[
S={n_k,n_{k+1},\ldots,n_i}
]

下一个音符为 (n_{i+1})。

令：

[
r=n_R(S)
]

即 (r) 是当前音符组中结束位置最大的音符。

定义当前组的音符主体右边界与下一音符起始位置之间的距离：

[
g=s(n_{i+1})-e(r)
]

允许在 (n_i) 与 (n_{i+1}) 之间进行切分，当且仅当同时满足：

[
g\ge D
]

以及：

[
e(r)+U(r)\le s(n_{i+1})-L(n_{i+1})
]

第二个条件等价于：

[
g\ge U(r)+L(n_{i+1})
]

因此，完整的切分条件可以合并写为：

[
g\ge
\max\left(D,\ U(r)+L(n_{i+1})\right)
]

### 6.3 扫描过程

初始化：

[
S={n_1}
]

依次处理 (n_2,\ldots,n_m)。

对于每个下一个音符 (n_j)：

* 若

[
s(n_j)-e(n_R(S))
\ge
\max\left(
D,,
U(n_R(S))+L(n_j)
\right)
]

则：

1. 输出分片 (P(S))；
2. 令

[
S={n_j}
]

* 否则，将 (n_j) 加入当前音符组：

[
S\leftarrow S\cup{n_j}
]

扫描结束后，输出最后一个分片 (P(S))。

该算法在每一个合法切分位置上立即切分。

---

## 7. 分片中音符的连续性

在按照 `position` 排序后的音符序列中，每个分片包含的音符必然构成一个连续子序列。

也就是说，不可能存在下标：

[
i<j<k
]

使得 (n_i,n_k) 属于同一个分片，而 (n_j) 属于另一个分片。

证明如下。

包含 (n_i,n_k) 的分片覆盖从不晚于 (s(n_i)) 到不早于 (e(n_k)) 的连续区间。由于：

[
s(n_i)<s(n_j)<s(n_k)<e(n_k)
]

所以 (s(n_j)) 位于该分片内部。

任何包含 (n_j) 的其他分片也必须覆盖 (s(n_j))，因而两个分片必然重叠，与分片互不重叠的要求矛盾。

因此，每个分片只能对应排序后音符序列的一个连续区间。

---

## 8. 唯一性

若同时满足以下条件：

1. 任意两个不同音符的 `position` 不相同；
2. 任意两个不同音符的

[
position+length
]

不相同；
3. 同一音符内任意两个音素的 `start` 不相同；

则最终得到的最大分片数结果唯一。

### 8.1 并列选择唯一

条件 1 保证每个音符集合中起始位置最小的音符唯一，因此 (n_L(S)) 唯一。

条件 2 保证每个音符集合中结束位置最大的音符唯一，因此 (n_R(S)) 唯一。

条件 3 保证音素排序唯一，因此 (c(n))、(L(n)) 和 (U(n)) 均唯一。

因此，任意给定的音符集合 (S) 所诱导的分片 (P(S)) 唯一。

### 8.2 最大分片数结果唯一

考虑顺序扫描算法维护的当前音符组 (S) 和下一个音符 (n_j)。

若切分条件成立，则当前组与后续音符可以形成两个合法分片。又因为：

[
s(n_j)-e(n_R(S))\ge D>0
]

当前组中所有音符的结束位置均严格小于 (s(n_j))。

而从 (n_j) 开始的任意后续音符的结束位置均严格大于 (s(n_j))。因此，当前组中的音符不可能决定任何后续分片的最大音符结束位置。

在此位置切分不会改变后续分片的右边界，也不会破坏后续任何合法切分。若某个合法结果未在此处切分，则可以增加该切分，从而使分片数量增加 1。因此，任何分片数量最大的结果都必须在此处切分。

反之，若切分条件不成立，则在当前音符组与下一个音符之间切分将违反以下至少一个条件：

* 音符主体间距不小于 (D)；
* 两个分片互不重叠。

因此，任何合法结果都不得在此处切分。

由左到右归纳可知，每一个切分位置均被唯一确定，故全部音符组及其所诱导的分片均唯一。

因此，在上述无并列条件下，不需要增加额外约束，输入音符集合能够唯一确定一组分片。

---

## 9. 存在并列情况时的规定

如果存在以下任一情况：

* 多个音符具有相同的 `position`；
* 多个音符具有相同的 `position + length`；
* 同一音符中多个音素具有相同的 `start`；

则允许在并列对象之间任意选择顺序或极值音符。

不同选择可能产生不同的：

* 前导非 onset 音素数量；
* 左填充量；
* 右填充量；
* 分片边界；
* 合法切分位置；
* 最终分片结果。

因此，在一般情形下，本规则只要求输出任意一个符合全部规则且分片数量最多的结果，不要求结果唯一。

具体实现可以使用音符 ID、音素 ID、输入顺序或其他稳定字段作为次级排序键，以获得确定性结果。

---

## 10. 时间单位

当前项目中的note.position和note.length的单位为tick，phoneme.start，填充基础量，填充附加量，分片间距的单位为毫秒，最终输出的分片的position和length的单位为tick。

内部计算的时候，统一使用毫秒作为时间单位。

1 beat = 480 tick。

beat与秒的转换需要借助TempoSequence。TempoSequence存储了工程中所有的曲速变更记号。初始曲速为120bpm，每个曲速变更记号表示从此记号开始到下一个曲速变更记号之间的曲速。

当前的note.position是相对于clip的位置。一个音符的绝对位置应当是note.position + clip.start。时间轴上的曲速变更记号的位置是绝对位置。
